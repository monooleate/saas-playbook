# 21 — Brand & Design System — Best Practices

> **Status:** Outline.

## Sources

- *Refactoring UI* — Wathan & Schoger.
- shadcn/ui, Radix UI, Tailwind docs.
- Linear's design writeups (public).
- Vercel Geist design system as inspiration.

## Practices

- Tokens before components.
- Steal from shadcn/ui; don't write primitives from scratch.
- One font for body, optionally one for display, never more than two.
- Don't ship a custom typeface unless you're already a brand. Use
  Inter, Geist, or system stack.
- Restraint beats variety. Fewer colors, fewer sizes, fewer shadows.
- Match the brand to the product's emotional register, not your taste.
