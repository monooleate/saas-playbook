# 20 — Pricing Strategy — Checklist

> **Status:** Outline.

## Discovery

- 10 target customers asked about price (Van Westendorp questions).
- Median acceptable price recorded.
- Competitor prices catalogued.

## Model

- Pricing model chosen: flat / per-seat / per-usage / hybrid.
- Reasoning written down.
- Anchor price set 10% above median.

## Tier design

- 3 tiers max for self-serve. (Enterprise can be "Contact us".)
- Each tier solves a different jobs-to-be-done.
- Middle tier is the obvious choice.
- Annual discount: 17–20%.

## Addon vs. tier decisions

- Each addon documented: why it's not in a tier.
- Default: bundle in tier; unbundle only if it raises ARPU
  meaningfully.

## Pricing page

- Public pricing.
- 3-column comparison.
- Monthly + annual toggle.
- Currency selector (or country-driven, see topic 03).
- Social proof on the page.

## Experiments

- A/B test framework set up (PostHog Experiments — see topic 05).
- Existing customers grandfathered for any test.
- One test running at a time.

## Raise prices

- Procedure documented for raising prices.
- Existing customers grandfathered or given 60-day notice.

(Each expanded in v0.2.)
